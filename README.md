### Важное обновление: автопрочистка self-backup старше 2 дней

Теперь CronJob сам очищает все backup-файлы старше двух суток прямо при запуске:

- До создания свежего архива с помощью команды:
  ```sh
  find /backup -name 'xui-backup-*.tar.gz' -mtime +2 -delete
  ```
- Дальше делается обычный архив.

**Что это значит:**
- На volume пода всегда хранятся только последние 2 дня backup-архивов.
- Старые backup-файлы физически удаляются, PV никогда не переполнится лишними копиями.
- Внешний отбор/копирование ты продолжаешь делать сам (ничего не поменялось по твоей схеме).

**Все изменения уже в файле**
```yaml
manifests/cronjob-backup.yaml
```

Механика полностью описана в Git-репозитории, всегда можно поменять число дней в find -mtime +N.
